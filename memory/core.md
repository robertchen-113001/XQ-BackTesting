## 一、專案願景

統一 MoneyDJ XQ 平台的回測報告系統——讓「選股中心」、「策略雷達」、「自動交易」三大平台的使用者，能在**同一個介面**下，以**一致的算法**分析策略績效，並透過豐富的分析工具（週期分析、因子分析）深化投資決策。

---

## 二、核心設計原則

### 2.1 算法一致性

- 所有平台（選股中心、策略雷達、自動交易、量化積木、因子投資、LocalBella）共用相同的**報表統計服務**
- 報酬率算法分兩種：**時間加權報酬率（TWR）** 與 **最大投入報酬率（MIR）**
- 交易數量分四種：**原始、等額、等量、等比**
- 計算公式詳見：`docs/prd/07-欄位計算.md`

### 2.2 UI 統一性

- 選股中心、策略雷達、自動交易三平台共用同一套 Web UI
- 介面不再出現任何平台專屬字眼，改以通用設計 + 動態顯示差異選項
- 一份 PRD 對應一個前端 Feature 模組（前端鏡像原則）

### 2.3 資料開放性

- 支援從 CSV 匯入使用者自訂的交易紀錄
- 支援匯出 `.BTReportNew`、`.xlsx`、`.csv` 三種格式
- 舊版 `.btreport` 維持相容性（可從舊入口匯入，僅支援舊版 UI）

## 三、技術架構

| 層次 | 技術 / 說明 |
|------|-------------|
| 前端 | Vite + React 18 + React Router v6（JSX，非 TypeScript）|
| 回測計算 | 後端報表統計服務（統一接口，各平台共用）|
| AP ↔ WebUI 通訊 | URL Query String 帶回測 ID；Sinker 機制回傳 ID |
| 儲存 | 本機硬碟永久保留，無數量上限，無自動刪除期限 |
| 版本控制 | Conventional Commits（`docs:`, `feat:`, `fix:` 等前綴）|
| 文件規範 | Markdown，存於 `docs/prd/`；自動化腳本用 Python，存於 `scripts/` |

## 四、核心限制（不可妥協）

| # | 限制 | 說明 |
|---|------|------|
| 1 | 等比不支援最大投入報酬率 | 交易數量選「等比」時，強制使用時間加權報酬率，不可切換 |
| 2 | 腳本交易數量僅限自動交易 | 選股中心、策略雷達不得顯示「腳本」選項 |
| 3 | 原始 Spec 唯讀 | `docs/reference/` 內容不可修改 |
| 4 | 版控不自動推送 | 絕對不執行 `git push`，最後推送由使用者手動完成 |
| 5 | 時間戳精度 | 進出場時間格式需含毫秒（`HH:MM:ss.fff`）|
| 6 | 計算精度 | 交易數量保留 6 位小數（超過捨去）；金額類一律 ROUNDDOWN 至整數 |
