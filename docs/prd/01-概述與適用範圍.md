## 1. 專案目標

| # | 目標 | 優先級 (MoSCoW) |
|---|------|----------------|
| 1 | 統一回測報告**算法**，確保各平台計算結果一致 | Must Have |
| 2 | 統一回測報告 **UI**，提供跨平台一致的使用者體驗 | Must Have |
| 3 | 支援交易數量類型：「原始」、「等額」、「等量」、「等比」 | Must Have |
| 4 | 支援報酬率算法：「時間加權」、「最大投入」 | Must Have |
| 5 | 支援匯入交易紀錄以產生回測報告 | Must Have |
| 6 | 強化回測報告分析功能（週期分析、因子分析等） | Should Have |

## 4. User Story 總覽

```
作為 [選股中心 / 策略雷達 / 自動交易] 的使用者，
我希望能在相同的介面上檢視回測報告，
以便快速比較不同平台的策略表現，而不需要學習不同的操作方式。
```

```
作為使用者，
我希望能自由切換「時間加權報酬率」或「最大投入報酬率」算法，
以便根據不同的分析目的解讀回測結果。
```

```
作為使用者，
我希望能匯入自訂的交易紀錄（CSV 格式）並產生回測報告，
以便分析在 XQ 系統以外產生的交易策略。
```

---

## 5. 技術架構說明

- **回測計算層**：適用所有 6 個產品，由統一的報表統計服務處理
- **回測 UI 層**：適用選股中心、策略雷達、自動交易（其餘產品暫不提供 UI）
- **溝通方式**：AP（XQ 應用程式）與 Web UI 透過 URL 帶回測 ID 互動，Web UI 再透過 Sinker 回傳 ID 給 AP
