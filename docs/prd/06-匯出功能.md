## 1. 匯出格式概覽

| 匯出類型 | 格式 | 命名規則 |
|---------|------|---------|
| 僅匯出交易紀錄 | `.csv` | `策略名稱_交易數量方式.csv`<br>例：`DIF-MACD從負翻正_等比.csv` |
| 完整匯出 | `.xlsx` | `策略名稱_交易數量_報酬率算法.xlsx`<br>例：`DIF-MACD從負翻正_等比_時間加權.xlsx` |
| 儲存報告 | `.BTReportNew` | 依回測 ID 命名 |

### 檔案大小限制

- 完整匯出 `.xlsx` 超過 **5 MB** 時：
  - 主檔案名稱不變
  - 以 5 MB 為單位依商品代碼順序**分割檔案**
  - 分割檔案命名：`策略名稱_交易數量_報酬率算法_part2.xlsx`

## 2. 僅匯出交易紀錄（CSV）

- 分頁內容與完整匯出中的「交易分析」分頁**相同**
- 欄位參見第 4.4 節「交易分析分頁」

> ℹ️ **User Story**
> 作為使用者，
> 我希望能快速匯出純交易紀錄 CSV，
> 以便在其他分析工具（如 Excel）中自行進行後續處理。

> 📐 **前端 Prototype 規劃（Vite）**
> - 元件：`ExportMenu.jsx`（下拉式選單，含三個匯出選項）
> - CSV 匯出：使用 `papaparse` 或自建 Blob 下載邏輯
> - 命名：前端動態組合檔名字串後觸發瀏覽器下載

## 3. 完整匯出 XLSX 分頁結構

| # | 分頁名稱 | 說明 |
|---|---------|------|
| 1 | 整體統計 | 整體回測統計數據 + 報酬率圖 + 失敗商品表格 |
| 2 | 設定 | 回測設定參數（基本資訊 + 交易設定）|
| 3 | 每日報表 | 每日損益與統計 |
| 4 | 商品統計表 | 各商品回測績效 |
| 5 | 交易分析 | 完整交易明細 |
| 6 | 族群透視 | 上市櫃 / 細產業 / 主題分類統計 |
| 7 | 週期分析(日) | 日頻率週期分析 |
| 8 | 週期分析(月) | 月頻率週期分析 |
| 9 | 週期分析(季) | 季頻率週期分析 |
| 10 | 週期分析(年) | 年頻率週期分析 |
| 11+ | 單商品統計 | 每個商品獨立一個分頁 |

## 4. 各分頁詳細規格

### 4.1 整體統計分頁

- 從 A1 開始寫入表格；標題列：**粗體 + 灰色底色**
- 欄位結構：

| 欄位 | 說明 |
|------|------|
| 項目 | 整體統計提供的所有欄位名稱 |
| 全部 \$ | 全部方向下的金額 |
| 全部 % | 全部方向下的報酬率 |
| 做多 \$ | 做多方向下的金額 |
| 做多 % | 做多方向下的報酬率 |
| 做空 \$ | 做空方向下的金額 |
| 做空 % | 做空方向下的報酬率 |

- 空一列，寫入報酬率圖：
  - 第一列（粗體）：「最大投入報酬率圖」或「時間加權報酬率圖」（依匯出時設定）
  - 第二列：報酬率 % / \$ 圖（含所有副圖）
- 空一列，寫入執行失敗商品表格：
  - 第一列（粗體）：「執行失敗商品」
  - 第二列：表格（標題列粗體 + 灰底）；欄位：商品名稱、狀態、說明

### 4.2 設定分頁

從 A1 開始寫入設定項目名稱（**粗體**），B1 開始寫入內容。

**基本資訊**：
- 回測執行時間
- 回測資料範圍
- 執行商品個數
- 商品名稱
- 策略名稱
- 進場設定（選股中心 / 策略雷達）
- 出場設定（選股中心 / 策略雷達）
- 交易腳本（自動交易）

**交易設定**：
- 初始資金
- 執行頻率
- 模擬逐筆洗價
- 交易費用
- 進場價格（選股 / 策略雷達：收盤或開盤；自動交易：觸發價 ± 檔）
- 出場價格（選股 / 策略雷達：收盤或開盤；自動交易：觸發價 ± 檔）
- 交易方向（選股中心 / 策略雷達）
- 最大持有期間（選股中心 / 策略雷達）
- 同時進場次數（選股中心 / 策略雷達）
- 每日部位歸零（自動交易）
- 單一商品最大部位（自動交易）
- 單一商品每日最多進場次數（自動交易）

### 4.3 每日報表分頁

從 A1 寫入表格，標題列粗體 + 灰底。

| 欄位 | 說明 |
|------|------|
| 日期 | — |
| 獲利 | 當日損益 |
| 總獲利 | 累計損益 |
| 最大投入金額 | — |
| 商品檔數 | — |
| 交易筆數 | — |

### 4.4 商品統計表分頁

從 A1 寫入表格，標題列粗體 + 灰底。
- 欄位：依照回測報告中商品統計表提供的欄位
- **每個商品佔兩列**：第一列為 \$（金額）；第二列為 %（報酬率）；無單位欄位填入第一列，第二列留空
- 商品名稱**合併儲存格**，可**超連結**至對應的單商品統計分頁

### 4.5 交易分析分頁

從 A1 寫入表格，標題列粗體 + 灰底。

| 欄位 |
|------|
| 商品名稱 |
| 序號 |
| 進場時間 |
| 進場方向 |
| 進場價格 |
| 出場時間 |
| 出場方向 |
| 出場價格 |
| 持有區間 |
| 交易數量 |
| 獲利金額 |
| 報酬率 |
| 累計獲利金額 |
| 累計報酬率 |
| 進場訊息 |
| 出場訊息 |

### 4.6 族群透視分頁

依序寫入「上市櫃」、「細產業」、「主題」三個區塊，每個區塊：
1. 第一列（粗體）：分類名稱
2. 第二列：圖片
3. 空一列
4. 表格（標題列粗體 + 灰底）：族群、報酬率、勝率、最大區間虧損
5. 空一列

### 4.7 週期分析各分頁（日 / 月 / 季 / 年）

每個分頁格式相同，以「週期分析(日)」為例：

**表格區**（從 A1 開始）：

| 欄位名稱 | 說明 |
|---------|------|
| 項目 | 各統計指標名稱 |
| \$ | 金額數值 |
| % | 報酬率數值（無單位項目留空）|

**統計項目**：
淨利 / 報酬率、勝率、最大區間獲利、最大區間虧損、獲利因子、總交易（日 / 月 / 季 / 年）數、獲利交易數、虧損交易數、毛利、毛損、最大單（日 / 月 / 季 / 年）獲利、最大單（日 / 月 / 季 / 年）虧損、平均單（日 / 月 / 季 / 年）金額變動、平均單（日 / 月 / 季 / 年）獲利金額變動、平均單（日 / 月 / 季 / 年）虧損金額變動、平均單（日 / 月 / 季 / 年）獲利虧損比

**圖片區**（每個圖形三列結構：標題 → 圖片 → 空列）：

| 頻率 | 圖片清單 |
|------|---------|
| 日 | 1日～31日（總計 %）、（平均 %）、（總計 \$）、（平均 \$）；星期一～五（總計 %）、（平均 %）、（總計 \$）、（平均 \$）；每一日（%）、每一日（\$）|
| 月 | 1月～12月（總計 %）、（平均 %）、（總計 \$）、（平均 \$）；每一月（%）、每一月（\$）|
| 季 | 第一季～第四季（總計 %）、（平均 %）、（總計 \$）、（平均 \$）；每一季（%）、每一季（\$）|
| 年 | 每一年（%）、每一年（\$）|

### 4.8 單商品統計分頁（每商品一頁）

- A1：「返回商品統計表」（**超連結**至商品統計表分頁）
- A2：表格（標題列粗體 + 灰底）：

| 欄位 |
|------|
| 項目（所有回測欄位名稱）|
| 全部 \$ / 全部 % / 做多 \$ / 做多 % / 做空 \$ / 做空 % |

- 空一列
- 明細表（第一列：「明細表」粗體；第二列：表格標題粗體 + 灰底）：

| 欄位 | 說明 |
|------|------|
| 交易序號 | 每筆進出場合為一個序號，**合併儲存格** |
| 交易日期 | 第一列進場；第二列出場 |
| 買賣別 | — |
| 交易價格 | — |
| 交易數量 | — |
| 獲利金額 | — |
| 報酬率 | — |
| 累計獲利金額 | — |
| 累計報酬率 | — |
| 持有區間 | — |
| 連續次數 | — |
| 訊息 | — |

> 📐 **前端 Prototype 規劃（Vite）**
> - 元件：`ExportMenu.jsx`（匯出入口）
> - 後端產出：XLSX 匯出建議由後端 API 產出二進位流，前端呼叫後觸發瀏覽器下載
> - 實作提示：若需純前端實作，可使用 `exceljs` 或 `xlsx` 套件
> - 注意：報酬率圖的嵌入（XLSX 內圖片插入）前端難以完全模擬，Prototype 階段可以佔位說明代替

## 5. User Story 總覽

```
作為使用者，
我希望完整匯出報告時，能根據目前的交易數量與報酬率算法設定進行匯出，
確保 Excel 內容與我在 UI 上看到的結果一致。
```

```
作為使用者，
當匯出的 XLSX 超過 5 MB 時，
我希望系統能自動拆分成多個檔案，並以 _part2 等方式命名，
以方便我管理大型回測結果。
```
