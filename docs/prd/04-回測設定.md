## 1. 概述

「回測設定」橫跨三個層次。三個層次互為輸入／輸出關係：系統參數提供啟動預設值，使用者於對話框微調後送出回測，最終結果在報告 UI 上完整呈現本次回測使用的參數。

### 1.1 三類設定範圍

| # | 設定類別 | 影響時機 | 章節 |
|---|---------|---------|------|
| ① | **系統參數設定**（S→K→P→R）| 啟動回測前的全域預設值 | §2 |
| ② | **執行回測對話框**| 使用者每次啟動回測時的當次設定 | §3 |
| ③ | **回測報告顯示**| 回測完成後在報告 Tab 6「交易設定」呈現本次參數 | §4 |

### 1.2 設定流向

三個層次的關係如下：

① **系統參數（S→K→P→R）** 儲存各平台的全域預設值（交易數量、報酬率算法、基準指標、疊加指標），各平台獨立儲存，互不影響。

② **執行回測對話框** 啟動時帶入系統參數預設值，使用者可於每次執行前微調（共用模組 + 平台差異模組）。

③ **回測報告 Tab 6「交易設定」** 在回測完成後完整呈現本次使用的參數（A 區設定資訊 + B 區腳本資料）。

### 1.3 適用平台

| 平台 | 系統參數獨立儲存 | 執行回測對話框 | 報告顯示 |
|------|----------------|---------------|---------|
| 選股中心 | ✓ | ✓ | ✓ |
| 策略雷達 | ✓ | ✓ | ✓ |
| 自動交易 | ✓ | ✓ | ✓ |

## 2. 系統參數設定（S→K→P→R）

**路徑**：系統 (S) → 設定 (K) → 系統參數 (P) → 回測報告 (R)

### 2.1 介面整合原則

| 項目 | 說明 |
|------|------|
| 介面統一 | 移除舊有「自動交易」專屬字眼|
| 適用平台 | 選股中心、策略雷達、自動交易 |
| 設定獨立 | 三個平台的設定值**各自獨立儲存**，互不影響 |
| 移除項目 | 不再提供「預設欄位顯示單位」、「表格數值顯示設定」、「報酬率曲線圖呈現設定」 |

### 2.2 參數設定（三平台各自獨立儲存，結構相同）

| 設定項目 | 選項內容 | 邏輯與限制 | Tooltip 說明 |
|---------|---------|-----------|-------------|
| 交易數量 | 腳本（僅自動交易）、等額（固定 10 萬元）、等量（固定 1 張）、等比 | 「腳本」選項僅在「自動交易」分頁顯示；等額金額與等量股數系統固定，不可調整 | 交易數量計算方式，影響回測報告數據呈現結果 |
| 報酬率算法 | 時間加權報酬率、最大投入報酬率、金額加權報酬率 | 若交易數量選「等比」，系統強制鎖定「時間加權報酬率」；「最大投入報酬率」與「金額加權報酬率」不可切換 | 報酬率提供三種算法：時間加權、最大投入、金額加權 |
| 基準指標 | 見下方 10 檔清單 | 清單由 server 端維護，使用者不可自訂；基準指標同時顯示於績效比較圖 | 基準指標用於計算 beta 等欄位，並自動顯示於績效比較圖上 |
| 疊加指標 | 任意標的（XQ 商品）/ 基準指標（即上方所選）/ 投組買進持有報酬率 / 不疊加 | 績效比較圖最多疊加**兩個**指標；選「基準指標」時自動帶入上方選定的標的 | 績效比較圖可疊加兩條參考線，在此設定預設選項 |

### 2.4 等比模式的報酬率算法限制

> ⚠️ 交易數量選「等比」時，三平台均強制鎖定「時間加權報酬率」；「最大投入報酬率」與「金額加權報酬率」灰底不可點擊。原因：等比模式無固定進場金額，金額類算法皆無法計算。

> ℹ️ **User Story**
> 作為使用者，當我選擇「等比」交易數量時，系統應強制將報酬率算法鎖定為「時間加權報酬率」，並將其他選項設為不可點擊，以避免在不支援的算法組合下產生錯誤的回測結果。

## 3. 執行回測對話框

### 3.1 對話框整體架構

對話框由「共用模組」與「平台差異模組」兩部分組成。共用模組在三平台規格與行為一致；平台差異模組依平台特性各自獨有。

**入口**：選股中心、策略雷達、自動交易主介面 → 執行回測按鈕。

**共用模組（§3.2，三平台一致）**
- 基本執行設定（執行頻率、價格資料、日期範圍）
- 交易設定（最大同時交易筆數、交易數量、報酬率算法、進場順序）
- 交易成本（股票 + 期貨手續費與交易稅）
- 槓桿（期貨保證金成數、股票融資 / 融券）
- 系統設定（Print 指令）

**平台差異模組（§3.3，依平台各自獨有）**
- 選股中心：作多 / 作空、市場別 + 執行範圍、進場設定、出場設定
- 策略雷達：執行商品、進場設定（含模擬逐筆洗價）、出場設定（股票 + 期貨停利停損）
- 自動交易：交易腳本、策略安全設定、預設委託參數、洗價觸發設定

### 3.2 共用模組規格

以下五個模組在三平台行為一致；欄位若預設值因平台而異會在備註欄標註。

#### 3.2.1 基本執行設定（共通部分）

| 欄位 | 資料型別 | 選項 / 範圍 | 預設值 | 備註 |
|------|---------|------------|--------|------|
| `frequency` | Enum | 日 / 分鐘等 | 日（自動交易為「1分鐘」） | 執行頻率 |
| `priceType` | Enum | 原始值 / 還原值 | 原始值 | 單選按鈕 |
| `startDate` | Date | yyyy/mm/dd | 2025/12/31 | 開始日期 |
| `endDate` | Date | yyyy/mm/dd | 2026/03/31 | 結束日期 |

**平台特有的基本執行欄位**詳見 §3.3（如 `tradeDirection`、`market`、`universe`、`targetSymbol`、`usMarketAll`、`simTick`、`dailyReset`、`preRunBars`）。

#### 3.2.2 交易設定（交易數量）

| 欄位 | 資料型別 | 選項 / 範圍 | 預設值 | 說明 |
|------|---------|------------|--------|------|
| `enableMaxConcurrent` | Boolean | Checkbox | True | 啟用最大同時交易筆數 |
| `maxConcurrentTrades` | Integer | 正整數 | 10 | 最大同時交易筆數，達上限後新訊號被忽略 |
| `volumeType` | Enum | 等額（固定 10 萬元）/ 等量（固定 1 張）/ 等比 / 腳本 | 等量 | 交易數量類型；等額/等量金額系統固定不可調整；「腳本」僅自動交易顯示 |
| `returnAlgorithm` | Enum | 時間加權報酬率 / 最大投入報酬率 / 金額加權報酬率 | 時間加權報酬率 | 等比交易數量時強制鎖定「時間加權報酬率」，其餘兩項 disabled |
| `entryOrderType` | Enum | 時間 / 股號 / 成交量 / 市值 / 股本 / 自訂 | 選股中心：股號；策略雷達 / 自動交易：時間 | 進場順序；「時間」不支援選股中心（無毫秒時間戳，見 §3.4）|

**Tooltip（最大同時交易筆數）**：
> 達到最大同時交易筆數上限後，新訊號將被略過。
> 範例：設為 10 筆時，最多同時持有 10 個部位。

**`enableMaxConcurrent` UX 行為**：
- 預設勾選（True）。勾選時 `maxConcurrentTrades` 欄位啟用，預設自動帶入 `10`，可編輯。
- 取消勾選時，`maxConcurrentTrades` 欄位 disabled（灰底不可輸入），數值保留但不送出。

**`volumeType` 與 `returnAlgorithm` 連動**：
- 選擇「等比」時，`returnAlgorithm` 強制鎖定「時間加權報酬率」，「最大投入報酬率」與「金額加權報酬率」呈灰底不可點擊。
- 原因：等比模式無固定進場金額，無法計算金額類算法。
- 從「等比」切換回其他交易數量模式時，`returnAlgorithm` 解鎖，恢復可選狀態。

**交易數量類型動態 UI**：

| 類型 | UI 顯示 | 固定值 | 支援平台 |
|------|---------|--------|---------|
| 等額 | 顯示標籤「10 萬元」（不可編輯）| 10 萬元 | 全平台 |
| 等量 | 顯示標籤「1 張」（不可編輯）| 1 張 | 全平台 |
| 等比 | 不顯示數值 | — | 全平台 |
| 腳本 | 不顯示數值 | — | **僅自動交易** |

> ⚠️ **等比模式限制**：選擇「等比」時，報酬率算法強制鎖定「時間加權報酬率」；「最大投入報酬率」與「金額加權報酬率」不可切換（等比模式無固定進場金額，無法計算金額類算法）。

## 4. 回測報告 UI 上顯示的回測參數

回測完成後，使用者透過報告詳細頁的 Tab 6「交易設定」回顧本次回測使用的全部參數。

### 4.1 顯示位置

| 項目 | 說明 |
|------|------|
| 入口 | 回測報告詳細頁 → Tab 6「交易設定」 |
| 內容區塊 | A 區「設定資訊」 + B 區「腳本資料」 |
| 對應元件 | `TradeConfigTab.jsx` |

### 4.2 A 區：設定資訊

#### 4.2.1 執行資訊（三平台共同）

| 資訊項目 | 格式 | 說明 |
|---------|------|------|
| 回測資料範圍 | `yyyy/mm/dd ~ yyyy/mm/dd` | 含執行模式說明（如：日 / 1分鐘）|
| 回測執行時間 | `yyyy/mm/dd hh:mm` | 含總時間、準備時間、回測時間 |
| 回測執行商品 | 商品範圍 + N（成功）+ N（失敗）| 點擊成功數跳轉商品統計；點擊失敗數彈出錯誤清單，提供「針對失敗商品重新回測」按鈕 |

#### 4.2.2 三平台共同顯示的設定欄位

| 欄位類別 | 對應 §3.5 欄位 |
|---------|---------------|
| 策略方向 | `tradeDirection`（選股中心）|
| 最大同時交易筆數 | `enableMaxConcurrent` + `maxConcurrentTrades` |
| 交易數量類型 | `volumeType` |
| 報酬率算法 | `returnAlgorithm` |
| 進場順序 | `entryOrderType` |
| 交易費用 | `stockFeeRate` / `stockTaxRate` / `futuresFee` / `futuresTaxRate` |
| 期貨保證金乘數 | `enableFuturesMargin` + `futuresMarginRate` |
| 進場價格設定 | `entryPriceType` |
| 出場價格設定 | `exitPriceType` |
| 每日部位歸零 | `dailyReset`（自動交易）|
| 安控參數 | `enableMaxPosition` / `maxPosition` 等（自動交易）|

#### 4.2.3 平台專屬顯示

| 平台 | 顯示內容 |
|------|---------|
| 選股中心 | 進場設定（腳本：`entryConditionName` + `param_*`）+ 出場設定（停利 / 停損 / 最大持有期間 / 其他腳本）|
| 策略雷達 | 進場設定（腳本：`entryScriptName` + `param_*` + 資料讀取設定）+ 出場設定（股票 + 期貨停利停損 / 最大持有期間 / 其他腳本）|
| 自動交易 | 腳本設定（顯示腳本進出場：`scriptName` + 腳本參數 + `defaultBuyBase` / `defaultSellBase` 等預設委託 + `immediateFill` 洗價觸發）|

### 4.3 B 區：腳本資料

| 項目 | 說明 |
|------|------|
| B1 頁籤 | 標題為腳本名稱（每個 `entryScriptName` / `exitScriptName` / `scriptName` 對應一個頁籤）|
| B2 修改日期 | 格式：`yyyy/mm/dd hh:mm` |
| B3 複製按鈕 | 一鍵複製腳本程式碼 |
| B4 程式碼顯示 | 顯示 XS 腳本程式碼（僅顯示，不可編輯）|

### 4.4 §3 欄位 → Tab 6 顯示對照表

| §3 模組 | Tab 6 顯示位置 | 全平台 / 限定平台 |
|--------|---------------|-----------------|
| 基本執行設定（共通）| A 區執行資訊 | 全平台 |
| 基本執行設定（平台差異）| A 區執行資訊 | 對應平台 |
| 進場設定 | A 區「進場設定」+ B 區腳本頁籤 | 選股中心、策略雷達 |
| 出場設定 | A 區「出場設定」+ B 區腳本頁籤 | 選股中心、策略雷達 |
| 交易腳本（AT）| A 區「腳本設定」+ B 區腳本頁籤 | 自動交易 |
| 策略安全設定 | A 區「安控參數」 | 自動交易 |
| 預設委託參數 | A 區「腳本設定」 | 自動交易 |
| 洗價觸發設定 | A 區「腳本設定」 | 自動交易 |
| 交易設定（共用，含 returnAlgorithm）| A 區共同欄位 | 全平台 |
| 交易成本 | A 區「交易費用」 | 全平台 |
| 槓桿 | A 區「期貨保證金乘數」 | 全平台 |
| 系統設定（Print）| **不顯示**（執行期偏好，非策略內容）| — |
