# PRD 04 — 回測報告：回測設定

> **版本**：v2.1 | **日期**：2026-04-17 | **狀態**：Draft

---

## 1. 概述

「回測設定」橫跨三個層次。三個層次互為輸入／輸出關係：系統參數提供啟動預設值，使用者於對話框微調後送出回測，最終結果在報告 UI 上完整呈現本次回測使用的參數。

### 1.1 三類設定範圍

| # | 設定類別 | 影響時機 | 章節 |
|---|---------|---------|------|
| ① | **系統參數設定**（S→K→P→R）| 啟動回測前的全域預設值 | §2 |
| ② | **執行回測對話框**| 使用者每次啟動回測時的當次設定 | §3 |
| ③ | **回測報告顯示**| 回測完成後在報告 「交易設定」呈現本次參數 | §4 |

### 1.2 設定流向

三個層次的關係如下：

① **系統參數（S→K→P→R）** 儲存各平台的全域預設值（交易數量、報酬率算法、基準指標、疊加指標），各平台獨立儲存，互不影響。

② **執行回測對話框** 啟動時帶入系統參數預設值，使用者可於每次執行前微調（共用模組 + 平台差異模組）。

③ **回測報告 「交易設定」** 在回測完成後完整呈現本次使用的參數（A 區設定資訊 + B 區腳本資料）。

### 1.3 適用平台

| 平台 | 系統參數獨立儲存 | 執行回測對話框 | 報告顯示 |
|------|----------------|---------------|---------|
| 選股中心 | ✓ | ✓ | ✓ |
| 策略雷達 | ✓ | ✓ | ✓ |
| 自動交易 | ✓ | ✓ | ✓ |

## 2. 系統參數設定（S→K→P→R）

**路徑**：系統 (S) → 設定 (K) → 系統參數 (P) → 回測報告 (R)

### 2.1 介面整合原則

| 項目 | 說明 |
|------|------|
| 介面統一 | 移除舊有「自動交易」專屬字眼|
| 適用平台 | 選股中心、策略雷達、自動交易 |
| 設定獨立 | 三個平台的設定值**各自獨立儲存**，互不影響 |
| 移除項目 | 不再提供「預設欄位顯示單位」、「表格數值顯示設定」、「報酬率曲線圖呈現設定」 |

### 2.2 參數設定（三平台各自獨立儲存，結構相同）

| 設定項目 | 選項內容 | 邏輯與限制 | Tooltip 說明 |
|---------|---------|-----------|-------------|
| 交易數量 | 腳本（僅自動交易）、等額（固定 10 萬元）、等量（固定 1 張）、等比 | 「腳本」選項僅在「自動交易」分頁顯示；等額金額與等量股數系統固定，不可調整 | 交易數量計算方式，影響回測報告數據呈現結果 |
| 報酬率算法 | 時間加權報酬率、最大投入報酬率、金額加權報酬率 ⚠️ | 若交易數量選「等比」，系統強制鎖定「時間加權報酬率」；「最大投入報酬率」與「金額加權報酬率」不可切換 | 報酬率提供三種算法：時間加權、最大投入、金額加權 |
| 基準指標 | 見下方 10 檔清單 | 清單由 server 端維護，使用者不可自訂；基準指標同時顯示於績效比較圖 | 基準指標用於計算 beta 等欄位，目前提供主要市場的 10 檔代表性指數。 |
| 疊加指標 | 自選商品（XQ 商品）/ 基準指標（即上方所選）/ 買進持有報酬率 / 不疊加 | 績效比較圖最多疊加**兩個**指標；選「基準指標」時自動帶入上方選定的標的；選「自選商品」時彈出 XQ 商品搜尋框 | 績效比較圖可疊加兩條參考線，在此設定預設選項 |

> ⚠️ **「金額加權報酬率」尚在評估中，不確定是否開發**。規格文件暫時保留此選項，實際是否上線待確認後更新。

### 2.3 R Tab 設定介面

R Tab 依三個平台各自獨立儲存設定，以分頁切換。每個平台的設定結構相同，包含三組：回測預設選項、績效圖基準指標、績效圖疊加指標。

### 2.4 等比模式的報酬率算法限制

> ⚠️ 交易數量選「等比」時，三平台均強制鎖定「時間加權報酬率」；「最大投入報酬率」與「金額加權報酬率」灰底不可點擊。原因：等比模式無固定進場金額，金額類算法皆無法計算。

## 3. 執行回測對話框

### 3.1 對話框整體架構

對話框由「共用模組」與「平台差異模組」兩部分組成。共用模組在三平台規格與行為一致；平台差異模組依平台特性各自獨有。

**入口**：選股中心、策略雷達、自動交易主介面 → 執行回測按鈕。

**共用模組（§3.2，三平台一致）**
- 基本執行設定（執行頻率、價格資料、日期範圍）
- 交易設定（最大同時交易筆數、交易數量、報酬率算法、進場順序）
- 交易成本（股票 + 期貨手續費與交易稅）
- 槓桿（期貨保證金成數、股票融資 / 融券）
- 系統設定（Print 指令）

**平台差異模組（§3.3，依平台各自獨有）**
- 選股中心：作多 / 作空、市場別 + 執行範圍、進場設定、出場設定
- 策略雷達：執行商品、進場設定（含模擬逐筆洗價）、出場設定（股票 + 期貨停利停損）
- 自動交易：交易腳本、策略安全設定、預設委託參數、洗價觸發設定

### 3.2 共用模組規格

以下五個模組在三平台行為一致；欄位若預設值因平台而異會在備註欄標註。

#### 3.2.1 基本執行設定（共通部分）

執行頻率、價格資料（原始值 / 還原值）、日期範圍為既有功能，欄位視覺位置見 §3.6 Wireframe。平台差異欄位（市場別、執行範圍、執行商品、逐筆洗價、每日歸零、預先執行筆數等）見 §3.3 各節。

#### 3.2.2 交易設定（交易數量）

| 欄位 | 資料型別 | 選項 / 範圍 | 預設值 | 說明 |
|------|---------|------------|--------|------|
| `enableMaxConcurrent` | Boolean | Checkbox | True | 啟用最大同時交易筆數 |
| `maxConcurrentTrades` | Integer | 正整數 | 10 | 最大同時交易筆數，達上限後新訊號被忽略 |
| `volumeType` | Enum | 等額（固定 10 萬元）/ 等量（固定 1 張）/ 等比 / 腳本 | 等量 | 交易數量類型；等額/等量金額系統固定不可調整；「腳本」僅自動交易顯示 |
| `returnAlgorithm` | Enum | 時間加權報酬率 / 最大投入報酬率 / 金額加權報酬率 | 時間加權報酬率 | 等比交易數量時強制鎖定「時間加權報酬率」，其餘兩項 disabled |
| `entryOrderType` | Enum | 時間 / 股號 / 成交量 / 市值 / 股本 / 自訂 | 選股中心：股號；策略雷達 / 自動交易：時間 | 進場順序；「時間」不支援選股中心（無毫秒時間戳，見 §3.4）；「自訂」觸發 XS 函數選擇器 |

**`entryOrderType`「自訂」UX 行為**：
- 選擇「自訂」時，系統彈出 XS 函數選擇器（同 XQ 腳本編輯器的函數插入入口）。
- 使用者選取排序用的 XS 函數後確認，函數名稱自動回填進場順序欄位。
- 此動線沿用 XQ 既有腳本函數元件，不需另行設計。

**Tooltip（最大同時交易筆數）**：
> 達到最大同時交易筆數上限後，新訊號將被略過。
> 範例：設為 10 筆時，最多同時持有 10 個部位。

**`enableMaxConcurrent` UX 行為**：
- 預設勾選（True）。勾選時 `maxConcurrentTrades` 欄位啟用，預設自動帶入 `10`，可編輯。
- 取消勾選時，`maxConcurrentTrades` 欄位 disabled（灰底不可輸入），數值保留但不送出。

**`volumeType` 與 `returnAlgorithm` 連動**：
- 選擇「等比」時，`returnAlgorithm` 強制鎖定「時間加權報酬率」，「最大投入報酬率」與「金額加權報酬率」呈灰底不可點擊。
- 原因：等比模式無固定進場金額，無法計算金額類算法。
- 從「等比」切換回其他交易數量模式時，`returnAlgorithm` 解鎖，恢復可選狀態。

**交易數量類型動態 UI**：

| 類型 | UI 顯示 | 固定值 | 支援平台 |
|------|---------|--------|---------|
| 等額 | 顯示標籤「10 萬元」（不可編輯）| 10 萬元 | 全平台 |
| 等量 | 顯示標籤「1 張」（不可編輯）| 1 張 | 全平台 |
| 等比 | 不顯示數值 | — | 全平台 |
| 腳本 | 不顯示數值 | — | **僅自動交易** |

> ⚠️ **等比模式限制**：選擇「等比」時，報酬率算法強制鎖定「時間加權報酬率」；「最大投入報酬率」與「金額加權報酬率」不可切換（等比模式無固定進場金額，無法計算金額類算法）。

#### 3.2.3 交易成本

| 欄位 | 資料型別 | 預設值 | 單位 |
|------|---------|--------|------|
| `stockFeeRate` | Float | 0.1425 | % |
| `stockTaxRate` | Float | 0.3 | % |
| `futuresFee` | Integer | 100 | 元/口 |
| `futuresTaxRate` | Float | 0.3 | % |

> ⚠️ 期貨交易稅 reference 顯示為 0.3%，與舊版 PRD 04 標示之 0.03% 不一致。本版本以 reference UI 為主（0.3%），如有實際差異請於後續校對。

#### 3.2.4 槓桿

| 欄位 | 資料型別 | 預設值 | 說明 |
|------|---------|--------|------|
| `enableFuturesMargin` | Boolean | True（勾選）| 啟用期貨保證金成數設定 |
| `futuresMarginRate` | Float | 13.5 (%) | 期貨保證金成數 |
| `stockFinancingRate` | Float | 60 (%) | 股票融資成數，**UI 保留欄位，暫不使用邏輯** |
| `stockMarginRate` | Float | 90 (%) | 股票融券保證金成數，**UI 保留欄位，暫不使用邏輯** |

#### 3.2.5 系統設定

`enablePrint`（啟動腳本內 Print 指令）為既有功能，欄位視覺位置見 §3.6 Wireframe；回測完成後不顯示於回測報告「交易設定」。

### 3.3 平台差異模組

#### 3.3.1 選股中心

**平台差異說明**：作多 / 作空切換（按鈕組）、市場別、執行範圍；進場設定（選股條件腳本 + 資料讀取筆數 + 腳本參數 + 最大同時進場次數 + 進場價格基準）；出場設定（出場腳本 + 停利 / 停損（股票）+ 最大持有時間 + 出場價格基準）。

以上欄位均為既有功能，欄位視覺位置見 §3.6.1 Wireframe。

#### 3.3.2 策略雷達

**平台差異說明**：執行商品（下拉 + 文字輸入）、美股全部時段；進場設定（進場腳本 + 資料讀取筆數 + 進場逐筆洗價 + 最大同時進場次數 + 進場價格基準）；出場設定（出場腳本 + 出場資料讀取筆數 + 出場逐筆洗價 + 停利 / 停損（股票 + 期貨）+ 最大持有時間 + 出場價格基準）。

以上欄位均為既有功能，欄位視覺位置見 §3.6.2 Wireframe。

#### 3.3.3 自動交易

**平台差異說明**：無作多 / 作空按鈕；無進出場條件設定，改以下列模組取代：
- **執行設定**：執行商品、逐筆洗價、每日部位歸零、美股全部時段、預先執行筆數
- **交易腳本**（獨有）：腳本名稱 + 動態腳本參數
- **策略安全設定**（獨有）：單一商品最大部位 / 每日最多進場 / 每分鐘最多交易
- **預設委託參數**（獨有）：買進 / 賣出價基準 + 偏移檔位 + 直接送出
- **洗價觸發設定**（獨有）：觸發即判斷成交

以上欄位均為既有功能，欄位視覺位置見 §3.6.3 Wireframe。舊版「初始資金」與「單邊交易費用」欄位已整合至共用模組，不再單獨顯示。

### 3.4 平台差異重點摘要

各平台的完整欄位分佈見 §3.6 Wireframe。下表僅列出**有特殊邏輯或跨平台差異**的關鍵規則：

| 規則 | 選股中心 | 策略雷達 | 自動交易 |
|------|---------|---------|---------|
